How it works

Five steps.
One laser prompt.

Designed for coding agents, agentic workflows, and extended chat sessions where context bloat becomes expensive.

00 — Your Prompt
It starts with you
You type normally. Sift intercepts before anything reaches the LLM — reading your intent, checking your history, and deciding what context is actually worth sending.
01 — Semantic Retrieval
Find what actually matters
Every message is embedded locally as a vector. When you send a query, Sift runs a similarity search and retrieves only the top relevant nodes — typically 2% of total context.
02 — Context Compression
Shrink without losing signal
Long context blocks are summarized. Redundancy is removed. Code is preserved verbatim. Dynamic token budgeting decides exactly how much to keep.
03 — Laser Prompt Assembly
One precise, high-signal prompt
System rules + user preferences + relevant context nodes + compressed query — assembled into the smallest possible prompt that produces the best possible answer.
04 — LLM
Your key. Your model.
The laser prompt is sent directly to your chosen provider — OpenAI, Anthropic, Gemini. Sift never sees your API key. Everything routes client-side.
05 — Response
Faster. Cheaper. Cleaner.
Output comes back optimized. Fewer tokens, sharper answers. Up to 60% reduction in output length — without losing the signal you actually needed.
